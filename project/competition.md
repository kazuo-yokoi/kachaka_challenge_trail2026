# Kachaka Simulation Challenge — 競技設計書

## 1. 競技の背景と目的

### 背景

Kachakaは家庭・施設向けの自律搬送ロボットであり、本競技ではそのシミュレーション環境を用いる。  
Kachakaには把持アームがなく、音声合成もシミュレーション環境では使用しないため、本競技で用いる機能は以下に限定される。

- 自律ナビゲーション（Nav2）
- カメラ（前面・後面・ToF深度）による環境認識
- LiDARによる障害物検知と地図活用
- 物体検出

### 目的

ROS2初学者が「動くものを作る」体験を通じて、以下の技術を段階的に習得する。

| フェーズ | 習得内容 |
|---|---|
| Task 1 | ROS2ノード作成、Nav2 ActionClient、座標系の理解 |
| Task 2 | 物体検出（YOLO）、状態機械の設計、カメラ処理 |
| Task 3 | 自律探索戦略、マルチクラス分類、タスクプランニング |
| Bonus | ToFカメラ、tf2による3D座標変換 |

---

## 2. 競技環境仕様

### シミュレーターと起動方法

```bash
# シミュレーション起動（warehouseワールド）
ros2 launch kachaka_utils launch_sim.launch.py map_name:=warehouse
```

### 使用ワールド

**メイン競技環境: `warehouse`**

| 要素 | 詳細 |
|---|---|
| 広さ | 約30m × 20m の倉庫 |
| 固定障害物 | 大型棚 × 3、小型棚 × 1、テーブル × 複数、椅子 × 複数 |
| ゴミオブジェクト | ボトル・カップ・缶（配置数はTask毎に異なる）|

### 使用可能なROS2トピック一覧

| トピック名 | 型 | 方向 | 用途 |
|---|---|---|---|
| `/kachaka/front_camera/image_raw` | `sensor_msgs/Image` | 受信 | 前面カメラ画像 |
| `/kachaka/front_camera/image_raw/compressed` | `sensor_msgs/CompressedImage` | 受信 | 圧縮版（推奨）|
| `/kachaka/back_camera/image_raw` | `sensor_msgs/Image` | 受信 | 後面カメラ画像 |
| `/kachaka/tof_camera/image_raw` | `sensor_msgs/Image` | 受信 | ToF深度画像 |
| `/kachaka/lidar/scan` | `sensor_msgs/LaserScan` | 受信 | 2D LiDARスキャン |
| `/kachaka/object_detection/result` | `kachaka_interfaces/ObjectDetectionListStamped` | 受信 | 物体検出結果（YOLO）|
| `/kachaka/manual_control/cmd_vel` | `geometry_msgs/Twist` | 送信 | 速度指令（手動制御）|
| `/tf` | `tf2_msgs/TFMessage` | 受信 | 座標変換（自己位置）|
| `navigate_to_pose` | Nav2 Action | 送受信 | ナビゲーション指令 |

### 制約事項

- 音声（VoiceManager）はシミュレーション環境では使用しない
- LLM（LLMManager）はシミュレーション環境では使用しない
- ロボットの物理的な操作（棚の移動など）は不可

## 3. Task 1 — ウェイポイントナビゲーション

### 概要

**難易度**: ★☆☆（初心者向け）  
**目的**: Nav2を使ってロボットを指定座標に移動させる基本動作を習得する

### ルール

1. 競技開始時にスタート地点（原点付近）にロボットを配置する
2. **5つのウェイポイント座標（x, y）** が事前に通知される
3. ロボットは **指定された順番** でウェイポイントを訪問する
4. 各ウェイポイントに **1.0m 以内** に到達した時点で「クリア」とみなす
5. 制限時間は **5分間**

### ウェイポイント例（warehouse環境）

```python
WAYPOINTS = [
    (5.0, 3.0),    # WP1: 棚エリア手前
    (-3.0, 5.0),   # WP2: 倉庫中央
    (8.0, -4.0),   # WP3: 奥エリア
    (-6.0, -2.0),  # WP4: 左エリア
    (0.0, 0.0),    # WP5: スタート地点に帰還
]
```

### スコア計算

```
スコア = 達成ウェイポイント数 × 20点
       + 時間ボーナス（達成時: 残り秒数 × 0.5点）
最大スコア: 100点 + 150点（時間ボーナス） = 250点
```

| 達成数 | 基本スコア |
|---|---|
| 5 / 5 | 100点 |
| 4 / 5 | 80点 |
| 3 / 5 | 60点 |
| 2 / 5 | 40点 |

### 評価ポイント

- 最短経路でウェイポイントを訪問できているか
- Nav2が失敗した場合のリカバリー処理があるか

## 4. Task 2 — ゴミ検出チャレンジ ★メインタスク★

### 概要

**難易度**: ★★☆（中級者向け）  
**目的**: YOLO による物体検出をナビゲーションと組み合わせ、指定座標でゴミを検出・記録するシステムを構築する

### シナリオ

倉庫内に **10個のゴミ**（ボトル）が配置されている。  
**ゴミの配置座標は事前に通知される**。ロボットは各座標に移動し、カメラでゴミを検出して記録する。

### ルール

1. 競技開始時に **10個のゴミの座標（x, y）** が通知される
2. ロボットは各座標に順番に移動し、カメラ画像でゴミを検出すること
3. ゴミを「発見」するには以下の条件を満たすこと
   - ゴミとの距離が **2.0m 以内**
   - カメラ画像内でYOLOがゴミを **信頼度 0.5 以上** で検出
4. 発見したゴミの情報を画像で出力すること
5. 同一ゴミの重複カウントは禁止（検出位置が 0.5m 以内の場合は同一とみなす）
6. 制限時間は **5分間**

### ゴミオブジェクトの種類

Task 2 では **ボトル（bottle）1種類のみ**。

| オブジェクト | YOLOクラスID | 説明 |
|---|---|---|
| ボトル | 39 | ペットボトルサイズの容器 |

### スコア計算

```
スコア = 発見数 × 10点
       + 時間ボーナス（制限時間内に全発見: 残り秒数 × 0.3点）
最大スコア: 100点 + 90点（時間ボーナス） = 190点
```

### 評価ポイント

- YOLOをROS2ノードに組み込み、検出結果をpublishできているか
- 検出ロジックの精度（誤検出・見落としの少なさ）
- ナビゲーション中に並行して検出できているか（状態機械の設計）

## 5. Task 3 — 完全探索・分類チャレンジ

### 概要

**難易度**: ★★★（上級者向け）  
**目的**: 未知の配置・複数種類のオブジェクトを自律探索し、正確に分類・記録するシステムを設計する

### シナリオ

Task 2との違い:
- ゴミの **配置数・位置・種類** がすべて非公開
- ゴミが **3種類** あり、それぞれを正しく分類する必要がある
- より広い探索戦略の検討が必要

### ルール

1. 事前に提供されるのは地図と初期位置のみ
2. 制限時間 **10分間** 以内に、できるだけ多くのゴミを発見・分類する
3. 発見・分類したゴミの情報を画像を含めて出力すること
4. 発見条件・重複禁止ルールはTask 2と同様

### ゴミオブジェクトの種類（3種類）

| オブジェクト | YOLOクラスID | 説明 |
|---|---|---|
| ボトル | 39 | ペットボトルサイズの容器 |
| カップ | 41 | コーヒーカップ等 |
| 缶 | 46 | 飲料缶 |

### スコア計算

```
発見スコア  = (発見数 / 全ゴミ数) × 100点
分類スコア  = (正しく分類した数 / 発見数) × 50点
時間ボーナス = 残り秒数 × 0.2点

スコア = 発見スコア + 分類スコア + 時間ボーナス
最大スコア: 100点 + 50点 + 120点 = 270点
```

### 評価ポイント

- 探索の網羅性（全エリアを効率的にカバーできているか）
- 分類精度（種類の判別が正確か）
- タスクプランニングの工夫（どんな探索順序・戦略を採用したか）

## 6. Bonus Task — 深度測定精度チャレンジ

### 概要

**難易度**: ★★★（任意参加）  
**目的**: ToFカメラを使ってゴミの3D座標を推定し、座標変換（tf2）を習得する

### ルール

1. Task 2または3で発見したゴミの **3D位置（x, y, z）をmap座標系** で推定し、publishする
2. 審判が実際の配置座標と比較して誤差を測定する

    ```
    トピック名: /garbage_finder/found_items_3d
    型: std_msgs/String（JSON形式）
    フォーマット: {"id": 1, "x": 3.2, "y": -1.5, "z": 0.05}
    ```

### スコア計算

```
1個あたりのスコア = max(0, 10 - 誤差[cm])点
スコア = Σ(各ゴミのスコア) / 発見数 × 10点（平均精度スコア）
```

| 誤差 | スコア（1個あたり）|
|---|---|
| 0cm（完璧）| 10点 |
| 5cm | 5点 |
| 10cm 以上 | 0点 |
